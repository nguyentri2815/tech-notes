Chịu trách nhiệm làm một Chuyên gia Kiến trúc Phần mềm (Software Architect). Tôi đang học Java theo lộ trình Event Sourcing / CQRS nâng cao. Hôm nay tôi hoàn thành bài học: [Ngày 1 — Tạo project Spring Boot + REST skeleton].

Hãy tạo cho tôi một file Tech Note bằng Markdown (.md) theo cấu trúc "Kiến trúc động" để tôi lưu lại cấu bối cảnh của ngày hôm nay. Yêu cầu file note phải đảm bảo các mục sau để khi tôi xem lại có thể nắm bắt nhanh trong 30 giây:

1. DASHBOARD TIẾN ĐỘ: Gồm trạng thái tổng quan, mục [⚡ ĐIỂM DỪNG HIỆN TẠI] mô tả chi tiết code đang dừng ở đâu, và mục [🎯 BƯỚC TIẾP THEO] của ngày mai là gì.
2. MÔ PHỎNG CÂY THƯ MỤC: Dạng khối văn bản thuần (code block text), đánh dấu rõ file nào mới xuất hiện hoặc bị refactor, giải thích ngắn gọn vai trò của file đó ngay bên cạnh bằng comment.
3. SƠ ĐỒ LUỒNG DỮ LIỆU (FLOW): Vẽ bằng Mermaid (Graph TD hoặc LR), phân chia rõ ranh giới các vùng (subgraph). Trong sơ đồ phải chỉ rõ [🔴 ĐIỂM THAY THẾ/NÂNG CẤP CHỐT YẾU].
4. CHI TIẾT SỰ DỊCH CHUYỂN LOGIC: So sánh trực quan bằng code block Java giữa "TRƯỚC ĐÓ" (bài cũ) và "BÂY GIỜ" (bài mới) tại file bị tác động mạnh nhất. Giải thích ngắn gọn lý do tại sao kiến trúc lại đổi như vậy.
5. QUY LUẬT ĐỌC LẠI 30 GIÂY: Hướng dẫn tôi mắt phải nhìn vào đâu trước, đâu sau khi mở lại file này để khôi phục ngữ cảnh (Context) nhanh nhất mà không bị rối.

Hãy viết ngắn gọn, trực quan, sử dụng thuật ngữ chuẩn Enterprise, không viết văn xuôi dài dòng.
